# design.md 模板与写作规则

design.md 是品牌视觉的 single source of truth：人读它来理解品牌，工具读它来翻译 theme.css。**改品牌永远先改 design.md，再同步 theme.css**。

## 核心纪律

1. **每个值可执行**——hex 码、字体名、具体元件名。写"米白色"是散文，写 `#F4F1E8` 才能翻译成 CSS
2. **每个色绑定用途**——孤立色块表没有意义，"`--paper` 背景主色"才有
3. **Do/Don't 来自真实反馈**——循环中用户每次否定都是一条候选红线，不要凭空编
4. **禁用项也要写**——明确不用什么（字体/颜色/手法）和用什么同样重要
5. **调性从行为回放**——从用户的点选、否决、反馈原话里提炼候选回放给用户挑，不让用户填空，更不凭空编

## 版式语法 (Layout Grammar)

<一句话：隔三米认出这个品牌靠什么——来自方向板轮的定稿>

**全局规则**（所有画幅适用，含内容页/每页 slides）:

- **明度结构**:<画面亮暗怎么分布，如"暗底，亮区集中不平铺">
- **焦点纪律**:<如"每页 1 个主焦点；档案信息（页码/章节）归入边缘条">
- **密度气质**:<仅定性三档之一：疏朗/适中/紧凑——品牌的默认气质，不是数值上限>
- **密度处理策略**:<内容多时怎么排不丢品牌感，如"多要点以分组卡片承载；边距下限不破；内容溢出优先拆页而非压缩边距">

**封面/题图构图**（封面、章节分隔页等低信息页适用——量化密度只允许写在本层）:

- <方向板的构图，如"左对齐单列；标题占上 1/3；元件固定右下角；内容实占 ≤ 1/2">

**适用范围注记**（以下注记原样写进 design.md 本段末尾）:

> 本版式语法由封面级小样提取并验证，定义品牌默认气质与低信息页构图。信息承载页（教学幻灯内页、图文详解等）的实际密度由内容决定——保持明度结构、对齐纪律、密度处理策略与签名手法即可维持品牌识别，量化密度限制不适用于信息承载页。

<构图规则必须按画幅分层写：全局规则仅纳入**在方向板全部场景（封面/幻灯内页/金句卡）上都成立的不变量**——对齐纪律、明度结构、密度气质、签名手法；只在封面场景成立的构图写进"封面/题图构图"层，不许上升为全局。**量化密度（面积比/元素数上限）只许出现在"封面/题图构图"层，写进全局规则是规格违例**——密度数值是内容的函数不是品牌的函数，由消费方引擎按实际内容决定。**写语法不写坐标**："信息入口从左上进"可以，"标题在左上角"不行——具体排版是消费方引擎（slides 生成等）在语法约束下的自由。本段不翻译进 theme.css（布局是消费方引擎的事）——它约束"排出来像不像"，不约束"排成什么"；每条都要可检验，不写"大气""舒服"这种散文>

## theme.css 翻译模板

design.md 定稿后按此模板翻译（详细规则见 SKILL.md ⑥）：

```css
/* <体系名> 品牌主题
 * 由 design.md 翻译生成 —— 改品牌先改 design.md，再同步此文件
 */
@import url("https://fonts.googleapis.com/css2?family=...&display=swap");

:root {
  /* —— 配色 token（与 design.md Palette 表一一对应）—— */
  --paper: #XXXXXX;
  --ink: #XXXXXX;
  --structure: #XXXXXX;
  --structure-rgb: R, G, B;      /* 需要透明度的色配 -rgb 伴生变量 */
  --accent: #XXXXXX;
  --accent-rgb: R, G, B;

  /* —— 字体 token —— */
  --font-title: "<标题字体>", "<中文回退>", serif;
  --font-body: "<正文字体>", serif;
  --font-mono: "<等宽字体>", monospace;
}

/* —— 暗色变体（可选；与 :root 同名 token 全量重声明，消费方切 data-theme 即换肤）—— */
[data-theme="dark"] {
  --paper: #XXXXXX;
  --ink: #XXXXXX;
  /* … :root 里的每个颜色 token 与 -rgb 伴生变量都要在此重声明 … */
}

/* —— 品牌元件（只写"长相"，布局/动画归消费方引擎）—— */
.stamp { color: var(--accent); border: 3px double var(--accent); /* … */ }
.card  { background: var(--off-white); box-shadow: 4px 4px 0 rgba(var(--structure-rgb), .12); }
```

## 暗色变体推导纪律

暗色不是重新设计，是定稿亮色板的**同色相翻译**。逐 token 推导：

- **底色**：沿亮底色相压暗到 L 10-16%（暖纸 → 暖黑，不许偷换成 `#111111` 中性黑，除非亮色板本身是冷调）；次底比底亮一档
- **文字**：用亮底的色调淡化出米白/灰白（不用纯白 `#FFFFFF`），与暗底对比度以 7:1 为目标
- **点缀/徽章**：色相不变，明度上提一两档，保证暗底上的辨识度
- **结构线/纹理**：反转为亮色低透明度（即混搭规则"暗色底 × 纸纹理"冲突的标准解法）
- **投影**：硬投影加深一档或换 1px 亮描边；柔光投影直接删（暗底上不可见）
- **伴生变量**：所有 `-rgb` 值随暗色重算
- **第三方官方色不动**

推导结果必须经用户确认（单区块 picker 出 2-3 个暗化方向候选，如暖黑 vs 深靛），不许静默交付。

自检清单：

- [ ] `:root` 之外 `grep -E "#[0-9A-Fa-f]{3,6}|rgba?\([0-9]"` 应为 0 命中
- [ ] 每个 design.md Palette 行都有对应变量
- [ ] 透明度用法全部走 `-rgb` 伴生变量
- [ ] 没有第三方官方色（那是消费方的事）
- [ ] 暗色块（若有）token 集与 `:root` 同名同数，且经过了用户确认
